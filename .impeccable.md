## Design Context

### Users
One person (the owner/developer) using this daily as a personal apartment-hunting workstation. Context: morning ritual, scanning new listings, making buy/skip decisions quickly. High stakes — this is a ₪3M+ purchase decision tool.

### Brand Personality
Premium real estate intelligence. Not a portal (Yad2/Madlan) — a personal command center. Three words: **Sharp. Confident. Curated.**

Emotional goal: feel like you're using a tool made for serious buyers, not a generic dashboard. Every session should feel like opening a Bloomberg terminal for apartments — authoritative, information-dense but refined.

### Aesthetic Direction
**Premium real estate** — think high-end property magazine meets modern data tool. Light mode. Warm off-whites and deep charcoal/slate (not navy blue), gold or warm amber accents rather than cold blue. Typography-forward: elegant serif or high-contrast sans for prices, refined reading experience for descriptions.

**Anti-references**: Yad2 (portal cluttered), generic AI dashboards (hero metrics + identical card grids + cold blue), corporate SaaS (navy + gray).

**Avoid**:
- Cold `#1e3a5f` navy — replace with warm deep tones
- Gray text on gray backgrounds
- Identical same-size card grid (AI tell #1)
- Stats bar as hero metrics at top (AI tell #2)
- System/default fonts

### Design Principles
1. **Curated, not catalogued** — fewer, better-presented listings feel more premium than a wall of cards
2. **Price is the protagonist** — the number should dominate visually; everything else supports it
3. **Warm confidence** — warm neutrals + one strong accent; never cold or corporate
4. **Information density without clutter** — all the data, arranged with editorial restraint
5. **Personal, not generic** — this is one person's tool; it can have an opinionated personality
